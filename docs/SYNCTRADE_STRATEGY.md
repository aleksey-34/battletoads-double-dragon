# Синхротрейд — Стратегия Hedge PnL Engine

## Суть

На MEXC Futures одновременно открываются **зеркальные позиции** на двух (или более) аккаунтах:
- **Мастер** → Long (прибыльная сторона)
- **Хедж** → Short (убыточная сторона)

Суммарный P&L ≈ 0 ± комиссии. Цель — **управляемое перемещение средств** между аккаунтами через P&L фьючерсных позиций.

## Параметры сессии

| Параметр | Описание | Рекомендация |
|---|---|---|
| **Символ** | Торговая пара | DOGEUSDT — высокая волатильность, низкий contractSize (10 DOGE), мелкий шаг |
| **Master Side** | Сторона мастера (Long/Short) | Long — проще для понимания |
| **Leverage** | Плечо | 5–10x — баланс между скоростью и ликвидацией |
| **Lot %** | Доля баланса в ордер | 30–50% — оставить запас на margin |
| **Целевой профит** | Когда закрыть | 5–20 USDT для быстрых сделок |

## Пошаговый цикл

```
1. Настроить мастер + хедж аккаунт (один раз)
2. Выбрать символ, leverage, lot%, target
3. Нажать "Открыть хедж-сессию"
4. Дождаться достижения целевого профита (авто-закрытие)
   — или закрыть вручную кнопкой "Закрыть"
5. Повторить
```

## Как быстро достичь таргета

### Скорость = Волатильность × Leverage × Размер позиции

| Фактор | Быстрее ↑ | Безопаснее ↓ |
|---|---|---|
| Leverage | 20–50x | 3–5x |
| Lot % | 80–100% | 20–40% |
| Символ | DOGE, SOL (волатильные) | BTC, ETH (стабильные) |
| Target | 2–5 USDT (маленький) | 20–50 USDT (большой) |

### Оптимальный профиль для "быстрых дел"

| Параметр | Значение |
|---|---|
| Символ | DOGEUSDT |
| Leverage | 20x |
| Lot % | 80% |
| Target | 10 USDT |
| Master Side | Long |

При балансе ~50 USDT и leverage 20x — notional ~800 USDT.
Движение DOGE на **1.25%** = профит мастера ~10 USDT.
DOGE часто двигается на 1–3% за 5–30 минут.

### Агрессивный профиль (максимальная скорость)

| Параметр | Значение |
|---|---|
| Символ | DOGEUSDT |
| Leverage | 50x |
| Lot % | 100% |
| Target | 5 USDT |
| Master Side | По тренду (Long в рост, Short в падение) |

⚠️ **Риск ликвидации** при leverage 50x — если цена пойдёт против мастера на ~2%, позиция ликвидируется.

## Важные моменты

1. **Комиссии**: ~0.02% taker × 2 стороны × 2 ордера (open + close) = ~0.08% от notional
2. **Фандинг**: При долгом удержании (>8ч) учитывать funding rate
3. **Ликвидация**: Чем выше leverage, тем ближе ликвидация — оставлять запас margin
4. **Направление рынка**: Если знаете куда пойдёт цена, ставьте мастера в эту сторону — профит быстрее

## Формула расчёта

```
Профит мастера = contracts × contractSize × (exitPrice - entryPrice) × direction
Убыток хеджа  ≈ -профит мастера
Чистый P&L    ≈ 0 - комиссии (≈ 0.08% × notional × 2)

Время до target ≈ targetUsdt / (notional × средняя_волатильность_за_минуту)
```

## Мониторинг

- **История сессий** — видно статус (open/closed/error), entry/exit, PnL
- **UPNL** — обновляется при рефреше (будет автообновление)
- **Авто-закрытие** — монитор проверяет каждые 10 секунд, закрывает при достижении таргета

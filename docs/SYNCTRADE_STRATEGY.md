# Синхро — Специализированная Алготрейдинг Стратегия

## Суть

На MEXC Futures одновременно открываются **зеркальные позиции** на двух (или более) аккаунтах:
- **Мастер** → прибыльная сторона
- **Хедж** → убыточная сторона

Суммарный P&L ≈ 0 ± комиссии. Цель — **управляемое перемещение средств** между аккаунтами через P&L фьючерсных позиций.

---

## Режимы работы

### 1. Ручной режим (Manual)

Один символ, одна сессия. Пользователь выбирает параметры, открывает/закрывает вручную.

| Параметр | Описание | Рекомендация |
|---|---|---|
| **Символ** | Торговая пара | DOGEUSDT — волатильность + мелкий contractSize |
| **Master Side** | Long/Short | Long — проще для понимания |
| **Leverage** | Плечо | 5–20x |
| **Lot %** | Доля баланса | 30–80% |
| **Целевой профит** | Когда закрыть | 5–20 USDT |

### 2. Авто-режим (SyncAuto Engine) ⚡

Движок автоматически сканирует рынок, выбирает лучшие пары, открывает много позиций параллельно, адаптирует нагрузку.

**Принцип**: загрузить баланс по максимуму (на ~2-3% от ликвидации), распределив по нескольким парам. На мастере — управляемый минус, на хедже — плюс.

## SyncAuto Engine — Архитектура

### Цикл работы

```
┌─ Scan (каждые 60с) ────────────────────────────────┐
│  1. fetchTickers() — все пары с volume/change        │
│  2. Фильтр: volume > 500K, change 0.5-15%,          │
│     contractSize < 5 USDT                            │
│  3. Скоринг: log(volume) × volatility_bonus          │
│  4. Открыть позиции до maxPairs                      │
└──────────────────────────────────────────────────────┘

┌─ Rebalance (каждые 15с) ───────────────────────────┐
│  1. Проверить все открытые позиции                    │
│  2. Расстояние до ликвидации < буфер?                │
│     → Закрыть пару (защита от ликвидации)            │
│  3. Мастер-позиция исчезла (ликвидация)?             │
│     → EMERGENCY: закрыть хедж моментально            │
│  4. Освободившиеся слоты → следующий scan заполнит   │
└──────────────────────────────────────────────────────┘
```

### Параметры конфигурации

| Параметр | По умолчанию | Описание |
|---|---|---|
| `maxPairs` | 6 | Макс. одновременных пар |
| `leverageRange` | [15, 30] | Адаптивное плечо (мин, макс) |
| `lotPercent` | 80 | % от баланса на все пары суммарно |
| `minVolume24h` | 500,000 USDT | Мин. оборот для ликвидности |
| `maxSpread` | 15% | Макс. change24h (фильтр пампов) |
| `liquidationBuffer` | 3% | Целевое расстояние до ликвидации |
| `scanIntervalMs` | 60,000 | Интервал сканирования |
| `rebalanceIntervalMs` | 15,000 | Интервал проверки нагрузки |
| `maxContractSizeUsdt` | 5 USDT | Макс. вес контракта (мелкие шаги) |

### Адаптивное плечо

Движок автоматически подбирает leverage в зависимости от волатильности пары:

| Волатильность 24h | Leverage |
|---|---|
| ~1% (низкая) | 30x (максимум) |
| ~5% (средняя) | 22x |
| ~10% (высокая) | 15x (минимум) |

**Формула**: `leverage = maxLev - (change24h - 1) / 9 × (maxLev - minLev)`

### Выбор пар (скоринг)

Каждая пара оценивается по формуле:
```
score = log10(turnover24h) × volatility_bonus × size_bonus

volatility_bonus = 1.5 если change24h ∈ [1%, 8%], иначе 1.0
size_bonus = 1.3 если contractSize < 1 USDT, иначе 1.0
```

**Фильтры отсечения**:
- turnover24h < 500K USDT → отсечение (неликвид)
- |change24h| > 15% → отсечение (памп/дамп)
- |change24h| < 0.5% → отсечение (слишком медленная)
- contractSize > 5 USDT → отсечение (грубые шаги)

### Направление мастера

Для каждой пары сторона мастера **рандомизируется** (50/50 long/short). Это диверсифицирует направленный риск — при движении рынка часть пар в плюсе, часть в минусе по мастеру, но хедж всегда зеркалит.

### Защита от ликвидации

1. **Превентивная**: Если расстояние до ликвидации < `liquidationBuffer / 2` → закрыть пару до ликвидации
2. **Аварийная**: Если мастер-позиция исчезла (ликвидирована) → моментально закрыть хедж-позицию
3. **Адаптивная**: Высоковолатильные пары получают меньшее плечо автоматически

#### Расчёт расстояния до ликвидации
```
liqDistance ≈ 1/leverage - 0.5% (maintenance margin)
currentDistance = |markPrice - entryPrice| / entryPrice
distanceToLiq = liqDistance - currentDistance

При leverage 20x: liqDistance ≈ 4.5%, buffer = 3%
→ Закрытие при unrealized loss > 1.5% от entry
```

### Ресурсоэффективность

- **1 вызов fetchTickers** (кешируется) → данные по всем парам
- **1 вызов fetchPositions** (без фильтра) → все позиции мастера за раз
- Rate limiting через Bottleneck per API key
- Scan каждые 60с, rebalance каждые 15с → ~6-8 API calls/мин в idle

## API эндпоинты

### Ручной режим
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/synctrade/:tenantId/execute` | Открыть сессию |
| POST | `/synctrade/:tenantId/close/:sessionId` | Закрыть сессию |
| GET | `/synctrade/:tenantId/sessions` | История сессий |

### Авто-режим
| Метод | Эндпоинт | Описание |
|---|---|---|
| POST | `/synctrade/:tenantId/auto/start` | Запустить движок |
| POST | `/synctrade/:tenantId/auto/stop` | Остановить (body: `{closeAll: true}`) |
| GET | `/synctrade/auto/status` | Статус, активные пары, ошибки |

### Пример запуска автомода
```json
POST /synctrade/47949/auto/start
{
  "maxPairs": 6,
  "leverageRange": [15, 30],
  "lotPercent": 80,
  "minVolume24h": 500000,
  "liquidationBuffer": 0.03
}
```

## Формулы

```
Профит мастера = contracts × contractSize × (exitPrice - entryPrice) × direction
Убыток хеджа  ≈ -профит мастера
Чистый P&L    ≈ 0 - комиссии (≈ 0.08% × notional × 2)

Бюджет на пару = (equity × lotPercent / 100) / maxPairs
Qty = (бюджет × leverage) / (price × contractSize)
```

## Мониторинг

- **История сессий** — статус (open/closed/error), entry/exit, PnL
- **UPNL** — обновляется при рефреше
- **Авто-закрытие** — монитор проверяет каждые 10 секунд, закрывает при достижении таргета
- **SyncAuto статус** — количество активных пар, циклов, ошибок

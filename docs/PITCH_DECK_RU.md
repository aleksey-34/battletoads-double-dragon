## Слайд 1 — Проблема

### Алгоритмическая торговля недоступна большинству

- **Высокий порог входа**: разработка торгового робота требует знаний в программировании, математике, работе с биржевым API. 95% ретейл-трейдеров не имеют этих навыков.
- **Нет валидации**: большинство готовых «ботов» на рынке не имеют прозрачной методологии бэктестирования и публикуют только «удобные» результаты.
- **Нет инфраструктуры**: даже если стратегия найдена — нужны VPS, мониторинг, аварийное закрытие позиций, управление ключами. Это операционная нагрузка, которая убивает трейдера.
- **Закрытые экосистемы**: существующие решения (3commas, Cryptohopper) — дорогие, непрозрачные, без возможности кастомизации.

---

## Слайд 2 — Решение

### BTDD Platform — Algorithmic Trading as a Service

Полноценная SaaS-платформа, которая даёт клиенту доступ к **проверенным алгоритмическим стратегиям** без необходимости разбираться в торговых деталях.

**Три режима для разных клиентов:**

| Режим | Для кого | Что получает |
|---|---|---|
| **Алгофонд** | Пассивный инвестор | Отдаёт депозит под управление. Видит эквити, статистику, запрашивает старт/стоп |
| **Клиент стратегий** | Продвинутый трейдер | Подключает свой API-ключ, выбирает стратегии, настраивает риск |
| **Администратор** | Управляющий / партнёр | Полный контроль платформы, sweep, publishing, клиенты, биллинг |

## Слайд 3 — Продукт

### Что уже построено (v2.0)

#### Торговый движок
- 3 типа стратегий: **DoubleDragon Breakout**, **StatArb Z-Score**, **ZigZag Breakout**
- Моно и синтетические инструменты (2 актива → один ticker цены)
- Trailing TP, Donchian Channel, Z-score с lookahead-free реализацией
- Изолированный runtime-сервис: торговля продолжается при рестарте API

#### Research & Backtesting
- Исторический sweep: **9 108 прогонов**, **0 ошибок**
- Robustness-фильтрация: **3 129 кандидатов** прошли PF/DD/WR/сделки фильтры
- Checkpoint/resume при длинных прогонах
- Параллельный grid-search по гиперпараметрам

#### SaaS-архитектура
- Multi-tenant изоляция по api_key
- Тарифные планы с лимитами
- Каталог офферов (TS) независим от runtime
- Audit log для критических действий

#### Frontend
- React + Ant Design SPA
- Dashboard, Research, SaaS Admin, Client Cabinet
- Поддержка RU / EN / TR

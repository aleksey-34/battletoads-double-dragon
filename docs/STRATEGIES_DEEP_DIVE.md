# Описание стратегий BTDD
**Дата:** 2026-04-02  
**Версия:** 2.0

---

## Обзор

В системе работают **3 типа торговых стратегий**, каждый из которых может быть запущен в **двух режимах рынка**:

| Тип | Код | Рыночный режим | Логика |
|---|---|---|---|
| DoubleDragon | `DD_BattleToads` | mono / synthetic | Пробой канала Дончиана |
| ZigZag | `zz_breakout` | mono / synthetic | Пробой канала Дончиана (аналогично DD) |
| StatArb Z-Score | `stat_arb_zscore` | **synthetic** (рекомендуется) | Возврат к среднему по Z-счёту |

**Моно-режим (mono)**: один актив, стандартная OHLCV-свеча.  
**Синтетический режим (synthetic)**: два актива, цена = `(baseCoef × base) / (quoteCoef × quote)`.

## 1. DoubleDragon Breakout (DD_BattleToads)

### 1.1 Концепция

DoubleDragon — трендовая стратегия пробоя канала Дончиана. Покупает при обновлении максимума за N баров, продаёт при обновлении минимума. Удерживает позицию, пока тренд продолжается, и фиксирует прибыль трейлинговым стопом от пика.

Разработана для работы на синтетических инструментах (пары активов), что снижает чувствительность к абсолютным движениям и увеличивает количество сигналов относительно работы на одном активе.

---

### 1.2 Параметры

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `price_channel_length` | 50 | Период канала Дончиана (баров) |
| `take_profit_percent` | 7.5 | Трейлинговый стоп от пика (%) |
| `detection_source` | `close` | Источник данных: `close` (по телу) или `wick` (по теням) |
| `interval` | `1h` | Таймфрейм |
| `long_enabled` | `true` | Разрешены лонги |
| `short_enabled` | `true` | Разрешены шорты |

---

### 1.3 Сигнал входа

**Close-mode** (detection_source = 'close'):
```
longBreakout  = current.close >= max(closes[-N..-1])
shortBreakout = current.close <= min(closes[-N..-1])
```

**Wick-mode** (detection_source = 'wick'):
```
longBreakout  = current.high  >= max(highs[-N..-1])
shortBreakout = current.low   <= min(lows[-N..-1])
```

Где `N = price_channel_length`.

> **Close-mode** — ждёт закрытия бара выше максимума. Меньше ложных срабатываний, позже вход.  
> **Wick-mode** — реагирует на касание уровня внутри бара. Ранний вход, но больше шума.

---

### 1.4 Trailing Take-Profit (трейлинговый выход)

```
Вход в long: entryPrice = current_close (± slippage)
             tpAnchor = current_close

На каждом следующем баре (long):
  tpAnchor = max(tpAnchor, current_close)         ← движется только вверх
  trailingStop = tpAnchor × (1 - tp_percent/100)

  Если current_close <= trailingStop:
    Закрыть позицию → reason: 'take_profit_long'
```

```
Вход в short: tpAnchor = current_close

На каждом следующем баре (short):
  tpAnchor = min(tpAnchor, current_close)         ← движется только вниз
  trailingStop = tpAnchor × (1 + tp_percent/100)

  Если current_close >= trailingStop:
    Закрыть позицию → reason: 'take_profit_short'
```

**Пример (long, tp = 7.5%):**
```
Вход:   100.0
Бар +1:  103.5 → anchor=103.5, stop=103.5×0.925=95.74
Бар +2:  108.0 → anchor=108.0, stop=108.0×0.925=99.9
Бар +3:  107.5 → anchor=108.0 (без изменений), stop=99.9
Бар +4:  112.0 → anchor=112.0, stop=112.0×0.925=103.6
Бар +5:  101.0 → 101 < 103.6 → ЗАКРЫТЬ. PnL = +12%
```

---

### 1.5 Stop-Loss (Донхиан-центр)

В дополнение к трейлинг-TP работает стоп по центру канала:

```
donchianCenter = (donchianHigh + donchianLow) / 2

Long: если current_close <= donchianCenter → закрыть (stop_loss_long_center)
Short: если current_close >= donchianCenter → закрыть (stop_loss_short_center)
```

Это быстрее срабатывает чем трейлинг при резком возврате в центр канала.

---

### 1.6 Signal Flip (разворот сигнала)

Если была открыта long-позиция и поступил short-сигнал:
1. Закрыть long → reason: `signal_flip`
2. Немедленно открыть short

---

### 1.7 Режимы работы

**Моно-режим (BTCUSDT, 4h, L12, TP5, Close):**
```
Инструмент: свечи BTCUSDT с Bybit
Логика: пробой 12-барного максимума BTCUSDT
Лонг: BTCUSDT BUY, Шорт: BTCUSDT SELL
Пример пар: BTCUSDT, ETHUSDT, BERAUSDT, ORDIUSDT
```
Хорошо работает на трендовых активах с выраженными паттернами пробоя.

**Синтетический режим (IPUSDT/ZECUSDT, 4h, L12, TP5, Close):**
```
Инструмент: синтетическая цена = IPUSDT / ZECUSDT
Логика: пробой 12-барного максимума соотношения IP/ZEC
Лонг:  BUY IPUSDT + SELL ZECUSDT
Шорт:  SELL IPUSDT + BUY ZECUSDT
```
Работает как парный трейд: ставка на относительную динамику двух активов. IP растёт быстрее ZEC → лонг синтетики. IP падает быстрее ZEC → шорт синтетики.

---

### 1.8 Примеры срабатывания

**Пример 1: Моно BERAUSDT 4h, L8, TP=7.5%, Close**
```
Контекст: BERA провёл 8 баров в диапазоне 4.5–5.2
Бар 9: close = 5.25 (новый 8-бар максимум)
  → LONG BERAUSDT
Следующие 4 бара: рост до 5.8
  anchor = 5.8, trailingStop = 5.8 × 0.925 = 5.365
Бар 14: откат до 5.30
  → close > stop, удерживаем
Бар 15: откат до 5.32
  → выше стопа, удерживаем
Бар 16: откат до 5.30, затем 5.27
  → close < trailingStop(5.365) → ЗАКРЫТЬ
  PnL: (5.27/5.25 - 1) × 100 = +0.38% — но anchor был 5.8 →
  trailingStop = 5.365; close = 5.30 < 5.365 → закрыть с PnL ≈ +1%
```

**Пример 2: Синт TRUUSDT/GRTUSDT 4h, L16, TP=10%, Wick**
```
Синт.цена = TRU/GRT
Контекст: соотношение в диапазоне 1.40–1.65
Пробой HIGH: wick нового бара = 1.68 > max(highs[-16]) = 1.66
  → LONG синтетики: BUY TRU + SELL GRT
Рост соотношения до 1.94
  anchor = 1.94, stop = 1.94 × 0.90 = 1.746
Откат: 1.70 < 1.746 → ЗАКРЫТЬ
PnL ≈ (1.70/1.68 - 1) × 100 ≈ +1.2% на нотионал
```

## 2. ZigZag Breakout (zz_breakout)

### 2.1 Концепция

ZigZag — разновидность стратегии пробоя канала, идентичная по алгоритму с DoubleDragon. На уровне кода используется та же функция `computeDonchianSignal()`. Отличие — в именовании, историческом контексте и возможных параметрах по умолчанию.

> **Технически**: `zz_breakout` и `DD_BattleToads` используют идентичный сигнальный алгоритм. Разница может быть только в параметрах конфигурации (меньший период, другой TP-процент, другой интервал).

---

### 2.2 Параметры (типичные для ZZ)

| Параметр | Типичное значение | Описание |
|---|---|---|
| `price_channel_length` | 5–16 | Короткий период для быстрой реакции |
| `take_profit_percent` | 2–5 | Меньший трейлинг для более частых сделок |
| `detection_source` | `close` или `wick` | |
| `interval` | `4h` | |

---

### 2.3 Особенности применения

ZigZag рекомендуется для:
- **Более активной торговли** с меньшим периодом (L=5–8) и TP=2–3%
- Рынков с высокой волатильностью и частыми пиками-впадинами
- Синтетических пар с выраженной mean-reversion компонентой

---

### 2.4 Примеры срабатывания

**Пример: Моно ORDIUSDT 4h, L5, TP=3%, Wick**
```
5-бар максимум highs: [0.420, 0.418, 0.415, 0.422, 0.419]
Wick нового бара: 0.426 (пробой max=0.422)
  → LONG ORDIUSDT
anchor = 0.426 → stop = 0.426 × 0.97 = 0.413
Бар +2: high = 0.44 → anchor = 0.44, stop = 0.44×0.97 = 0.427
Бар +3: close = 0.425 < 0.427 → ЗАКРЫТЬ
PnL ≈ (0.425/0.426 - 1) ≈ -0.2%  ← срабатывание на 3% откате от пика
```

**Пример: Short Синт STXUSDT/INJUSDT 4h, L8, TP=4%**
```
Соотношение STX/INJ пробивает 8-бар минимум вниз
  → SELL STX + BUY INJ
Пебой был ложным → соотношение возвращается вверх
  anchor (min) = 0.485, stop = 0.485 × 1.04 = 0.504
Когда ratio >= 0.504 → закрыть
Убыток: -1.5% на нотионал (кандидат исключается свипом)
```

## 3. StatArb Z-Score (stat_arb_zscore)

### 3.1 Концепция

Статистический арбитраж на основе Z-счёта. Стратегия предполагает, что ценовое соотношение двух коррелированных активов (синтетическая цена) стационарно и возвращается к своему историческому среднему. Когда отклонение становится статистически значимым (Z > threshold), открывается позиция против тренда, ожидая возврата.

**Идеальные кандидаты**: пары активов из одной ниши (DeFi, Layer-2, oracles и т.д.) — ORDI/ZEC, IP/ZEC, BERA/ZEC, GRTUSDT/INJUSDT.

---

### 3.2 Параметры

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `price_channel_length` | 120 | Окно расчёта mean и stddev (баров) |
| `zscore_entry` | 2.0 | Порог входа по Z-счёту |
| `zscore_exit` | 0.5 | Порог выхода (возврат к среднему) |
| `zscore_stop` | 3.5 | Стоп-лосс по Z-счёту (дальнейшее отклонение) |
| `take_profit_percent` | 0 | Не используется (выход по z-score) |

---

### 3.3 Расчёт Z-счёта

```python
# Последние N баров (price_channel_length)
window = closes[-N..-1]  # N значений до текущего
mean   = sum(window) / N
sigma  = sqrt(sum((x - mean)^2 for x in window) / N)

# Текущий Z-счёт
Z = (current_close - mean) / sigma
```

---

### 3.4 Правила входа

```
SHORT (цена слишком высокая, ожидаем падение):
  Z >= +zscore_entry
  → Цена выше среднего на ≥ 2σ
  → Открыть SELL (для синтетики: SELL base + BUY quote)

LONG (цена слишком низкая, ожидаем рост):
  Z <= -zscore_entry
  → Цена ниже среднего на ≥ 2σ
  → Открыть BUY (для синтетики: BUY base + SELL quote)
```

---

### 3.5 Правила выхода

**Mean-revert exit (целевой выход):**
```
Long close:  Z >= -zscore_exit  (Z вернулся к нулю с нижней стороны)
Short close: Z <=  zscore_exit  (Z вернулся к нулю с верхней стороны)
```

**Stop-loss exit (продолжение тренда):**
```
Long stop:  Z <= -zscore_stop  (ДАЛЬШЕ ушёл вниз, ошиблись)
Short stop: Z >=  zscore_stop  (ДАЛЬШЕ ушёл вверх, ошиблись)
```

---

### 3.6 Визуализация логики

```
         +zscore_stop (+3.5σ)  ← SHORT стоп (cut loss)
         +zscore_entry (+2.0σ) ← SHORT вход
━━━━━━━━━────────────────────────────────────────────
         mean (0σ)              ← SHORT exit / LONG exit
━━━━━━━━━────────────────────────────────────────────
         -zscore_entry (-2.0σ) ← LONG вход
         -zscore_stop  (-3.5σ) ← LONG стоп (cut loss)

Пример Z-curve:
  Т=0:  Z=1.1  → нет сигнала
  Т=5:  Z=1.8  → нет сигнала (меньше 2.0)
  Т=8:  Z=2.1  → ОТКРЫТЬ SHORT (Z >= +2.0)
  Т=14: Z=1.5  → удерживаем
  Т=20: Z=0.4  → ЗАКРЫТЬ SHORT (Z <= +0.5, mean-revert exit)
```

---

### 3.7 Режимы работы

**Синтетический (рекомендуемый):**
```
Пара: IPUSDT/ZECUSDT (IP-ZEC соотношение)
Логика: IP и ZEC — оба layer-1/экосистема токены.
        Их соотношение исторически устойчиво.
        При аномальном росте IP относительно ZEC:
          Z > 2.0 → SELL IP + BUY ZEC (шорт синтетики)
        Ожидаем: IP откатится или ZEC подтянется
        Выход при Z < 0.5
```

**Моно (менее типичный):**
```
Пара: BERAUSDT (один актив против времени)
Логика: текущая цена значительно выше 120-бар среднего
        Z > 2.0 → SHORT BERA (ожидаем откат к среднему)
Риск: BERA может продолжить рост (трендовый рынок ломает stat_arb)
```

---

### 3.8 Примеры срабатывания

**Пример 1: Успешный — ORDIUSDT/ZECUSDT 4h, L72, ZE=2.0, ZX=0.5, ZS=3.5**
```
Ситуация: ORDI резко вырос относительно ZEC после листинга
window 72 баров: mean=0.0412, sigma=0.0031
Текущая синт.цена: 0.0476
Z = (0.0476 - 0.0412) / 0.0031 = 2.06 → SHORT
  → SELL ORDIUSDT + BUY ZECUSDT

Следующие 15 баров: ORDI консолидируется, ZEC восстанавливается
Синт.цена снижается до 0.0429
Z = (0.0429 - 0.0412) / 0.0031 = 0.55 → ещё рано

Бар +18: синт.цена = 0.0425
Z = (0.0425 - 0.0412) / 0.0031 = 0.42 ≤ 0.5 → ЗАКРЫТЬ
PnL = (0.0476 - 0.0425) / 0.0476 × notional ≈ +10.7% на нотионал
```

**Пример 2: Стоп — GRTUSDT/INJUSDT 4h, L48, ZE=1.75, ZS=3.0**
```
Ситуация: GRT показал аномальный рост, INJ скорректировался
mean=1.82, sigma=0.18
Z = (2.25 - 1.82) / 0.18 = 2.39 → LONG (Z ≤ -1.75 — нет, это неверно)

Корректно: если Z = -2.39 (GRT упал):
  → LONG: BUY GRT + SELL INJ
Z продолжает падать до -3.15:
  -3.15 ≤ -3.0 (zscore_stop) → ЗАКРЫТЬ LONG (stop)
  Финансирование: GRT продолжил тренд вниз
  Убыток: -(3.15-1.75)/1.75 × ... ≈ -8% на нотионал
  
Вывод: стоп защищает от случаев, когда арбитраж ломается из-за фундаментала
```

## 4. Сравнение стратегий

| Характеристика | DoubleDragon (DD) | ZigZag (ZZ) | StatArb (SA) |
|---|---|---|---|
| **Тип** | Трендовая | Трендовая | Контртрендовая |
| **Сигнал** | Пробой N-бар max/min | Пробой N-бар max/min | Z-отклонение > ±Nσ |
| **Выход-TP** | Трейлинговый стоп | Трейлинговый стоп | Возврат Z к 0 |
| **Выход-SL** | Донхиан-центр | Донхиан-центр | Z пересекает stop |
| **Позиция** | Долгая (дни-недели) | Средняя (часы-дни) | Средняя (дни) |
| **Лучший рынок** | Тренд + волатильность | Частые колебания | Боковик с корреляцией |
| **Риск** | Ложный пробой | Ложный пробой | Тренд ломает парность |
| **Рек. режим** | Mono + Synthetic | Mono + Synthetic | **Synthetic** |
| **Рек. интервал** | 4h–1d | 1h–4h | 4h–1d |
| **Рек. период** | 12–36 | 5–16 | 48–120 |

## 5. Синтетический инструмент

### 5.1 Формула

```
SyntheticPrice = (baseCoef × basePrice) / (quoteCoef × quotePrice)

По умолчанию: baseCoef = quoteCoef = 1
  SyntheticPrice = base / quote
```

### 5.2 Построение OHLC

Для каждого timestamp t:
```
subCandle.open  = (baseCoef × base.open)  / (quoteCoef × quote.open)
subCandle.close = (baseCoef × base.close) / (quoteCoef × quote.close)
subCandle.high  = max(open, close, base.high/quote.low, base.low/quote.high)
subCandle.low   = min(open, close, base.high/quote.low, base.low/quote.high)
```

Агрегация по таргет-интервалу (4h = 4 часовых свечи):
- bucket открывается по первой sub-свече
- high = max all sub-high
- low = min all sub-low
- close = last sub-close
- volume aggregated

### 5.3 Преимущества

1. **Снижение рыночного риска** — пар-трейд нейтрализует движение крипто-рынка в целом
2. **Лучшая стационарность** — соотношение двух связанных активов стационарнее, чем абсолютная цена
3. **Больше возможностей** — N рынков × M рынков вариантов пар вместо N рынков

### 5.4 Исполнение синтетики

```
Long synthetic (base растёт vs quote):
  1. BUY base_qty базового актива
  2. SELL quote_qty котировочного актива

Short synthetic (base падает vs quote):
  1. SELL base_qty базового актива
  2. BUY quote_qty котировочного актива

Пропорция: baseQty × basePrice ≈ quoteQty × quotePrice ≈ targetNotional / 2
Квантование: iterative best-fit по qtyCandidates
```

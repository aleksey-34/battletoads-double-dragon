# ПЛАН: Клиентский кабинет (ЛК) — Доработка v3

Дата: 2026-04-02

## Текущее состояние

### Что есть:
- Админ видит витрины Алгофонда и Стратегий с карточками + чартами → OK
- Клиент Алгофонда видит: базовое инфо (read-only), слайдер риска, preview эквити, кнопки start/stop
- Клиент Стратегий видит: аналогично но ограниченно
- Бектест-дровер открывается с TS-бектестом и слайдерами risk/frequency
- Тарифы (plans) с capabilities: backtest, monitoring, settings, apiKeyUpdate, startStopRequests

### Что сломано / отсутствует:
1. **Мониторинг счёта** — ЛК отсылает на `/positions`, но там нет чарта эквити клиента
2. **Список API-ключей** — клиент не видит свои ключи и не может управлять ими
3. **Витрина у клиента стратегий** — пусто (нет published strategy offers с клиентской стороны)
4. **Витрина у клиента Алгофонда** — карточки есть но без чартов эквити
5. **Предпросмотр портфеля** — лишний, удалить
6. **Карточки в ЛК должны = карточки на витрине админа** — сейчас клиент видит упрощённый read-only вариант

#### A2. Витрина клиента Стратегий — показать published карточки

### БЛОК B: Бектест в ЛК клиента

#### B1. Упрощённый бектест для клиента Алгофонда
- При клике на карточку ТС → открывается дровер
- Простое описание ТС (текст)
- ТОЛЬКО крутилка "Риск" (множитель к базовому размеру позиции)
- 1 чарт эквити — чистый, без лишнего
- Настройка риска влияет на эквити (пересчёт)
- Сохраняется как мультипликатор к карточке у клиента

#### B2. Бектест для клиента Стратегий
- При клике на карточку → дровер
- Крутилка "Риск" + крутилка "Частота сделок"
- 1 чарт эквити
- Настройки пересчитывают эквити
- Сохраняются у клиента

---

### БЛОК C: Составление TS

#### C1. TS-композер для клиентов Стратегий
- Клиент может создать свою ТС из доступных оферов
- По тарифу: ограничение на кол-во стратегий в ТС
- Может запустить бектест своей ТС
- Созданная ТС сохраняется в "Оферы и клиенты" с пометкой "клиентская"
- НЕ публикуется на витрину Алгофонда, но возможность публикации есть

#### C2. TS-композер для Админа
- Когда Админ составляет ТС из оферов → сохраняется в "Оферы и клиенты" 
- Без автопубликации на витрину Алгофонда
- С возможностью опубликовать вручную

#### C3. Место для сохранённых ТС в интерфейсе клиента Стратегий
- Раздел "Мои ТС" в ЛК клиента стратегий
- Список сохранённых ТС с быстрым бектестом

---

### БЛОК D: Ограничения и тарифы

#### D1. 1 TS на 1 API-ключ 
- Запрет запуска нескольких TS одновременно на одном ключе
- Backend: проверка при активации TS
- Frontend: блокировка кнопки "Запустить" если TS уже работает на ключе

#### D2. Кросс-блокировка Алгофонд ↔ Стратегии на одном ключе
- Клиент Стратегий не может запустить Алгофонд на том же ключе и наоборот
- Backend: проверка product_mode при привязке ключа
- Frontend: валидация + предупреждение

#### D3. Расширение тарифа (Алгофонд + Стратегии)
- Если клиент Алгофонда хочет ещё и стратегии — нужен расширенный тариф
- Добавить plan с combined capabilities
- Или addon-механизм к существующему плану

### БЛОК E: UX-улучшения

#### E1. Удаление стратегий при отключении клиента

## Схема данных для сохранённых TS

```
trading_systems:
  + owner_type: 'admin' | 'client'    -- кто создал
  + owner_tenant_id: number | null     -- id клиента-создателя (null для админских)
  + published_storefront: boolean      -- опубликована ли на витрину алгофонда
  + source: 'composer' | 'offer'       -- откуда создана
```

## Кросс-блокировка ключей

Текущая таблица `tenants`: у каждого tenant есть `product_mode` и `assigned_api_key_name`.
Проверка: при привязке ключа убедиться что нет другого tenant с другим product_mode на этот же ключ.
Исключение: tenant может иметь расширенный тариф (addon) для обоих режимов.
